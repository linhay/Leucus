# Infinite Canvas M1（网点背景 + 空节点卡片）

## 背景
- 当前阶段只做画布基础能力，不做节点类型与终端绑定。
- 画布能力拆分为独立 SPM target，避免和终端逻辑耦合。

## 范围
- 独立 target：`InfiniteCanvasKit`
- 独立 target：`FolderCard`（承载文件夹卡片浏览视图）
- 无限画布视口能力：平移、缩放、坐标转换
- 背景样式：网点背景
- 节点形态：空卡片（仅占位，类型待定）
- 交互增强：右键菜单可添加终端类型卡片
- 交互增强：右键菜单可添加文件夹类型卡片
- 交互增强：终端卡片与文件夹卡片支持右键互相切换
- 交互增强：支持拖入文件夹创建终端类型卡片（并携带工作目录）
- 交互增强：支持拖入文件夹创建文件夹类型卡片（并携带工作目录）
- 终端卡片归类：统一归到“文件夹”大类，并展示大类标签
- 路径抽象：节点标题与大类标签基于 `STFilePath` 生成
- 文件夹浏览：文件夹卡片可浏览当前目录内容，并支持进入子目录与返回上级目录
- 可读性约束：文件夹卡片内容区采用高对比主题（深色背景 + 高对比文字/图标 + 选中高亮）
- 卡片整理：支持一键自动整理卡片为平衡网格布局（避免重叠）
- 卡片对齐：通过画布右下角“对齐网格”高对比悬浮按钮触发位置吸附到网点网格
- 卡片弹出：支持将终端/文件夹卡片展开为独立窗口（窗口可独立关闭）
- 状态持久化：自动保存并在下次启动恢复画布（viewport、节点、选中态）
- 最小尺寸折叠态：节点在最小值时只显示类型图标、名称和一行摘要
- 边缘交互反馈：鼠标靠近可拉伸边/角时，高亮对应边框
- 终端重排策略：节点缩放过程中冻结终端 reflow，松手后统一刷新一次，减少 Ghostty 残影

## 非目标
- 不做节点类型系统
- 不做节点拖拽编辑、连线、持久化（M2/M3）
- 不做终端渲染挂载

## BDD 验收场景
1. Given 用户打开 Example
   When 应用完成渲染
   Then 可见网点背景和至少 1 张空卡片

2. Given 用户在画布上拖动/滚动
   When 触发平移
   Then 画布内容可连续移动且无跳变

3. Given 用户进行缩放（pinch 或 Command+滚轮）
   When 缩放围绕锚点计算
   Then 锚点对应世界坐标保持稳定

4. Given 连续极限缩放
   When 超出约束
   Then scale 被 clamp 在最小/最大范围内

5. Given 用户在画布空白区域右键
   When 选择“添加终端卡片”
   Then 在点击位置新增 `terminal` 类型节点并自动选中

6. Given 用户将一个或多个文件夹拖入画布
   When 拖拽释放到画布
   Then 为每个文件夹创建一个 `terminal` 类型节点，并将该文件夹路径写入节点工作目录

7. Given 用户在画布空白区域右键
   When 选择“添加文件夹卡片”
   Then 在点击位置新增 `folder` 类型节点并自动选中

8. Given 用户将一个或多个文件夹拖入画布
   When 拖拽释放到画布
   Then 为每个文件夹创建一个 `folder` 类型节点，并将该文件夹路径写入节点工作目录

9. Given 用户关闭并重新启动应用
   When 画布初始化
   Then 自动恢复上次保存的 viewport、节点列表与选中节点

10. Given 用户将鼠标移动到卡片可拉伸边缘附近
   When 指针进入边/角命中区域
   Then 卡片显示对应边框高亮，提示可拉伸方向

11. Given 用户正在拖拽调整终端卡片宽高
   When 拖拽尚未结束
   Then 终端视图不做实时 reflow，仅在松手后统一刷新

12. Given 鼠标位于终端卡片上方并触发滚轮
    When 终端滚动已到边界或无可滚动内容
    Then 滚轮事件应透传给画布，画布继续平移

13. Given 鼠标位于终端卡片上方并触发滚轮
    When 终端仍有可滚动内容且滚动方向可继续
    Then 应优先滚动终端内容，不应先触发画布平移

14. Given 画布上存在多个窗口卡片
    When 用户选中任意一个窗口
    Then 该窗口应提升到最前层显示（包含终端内容层）

15. Given 鼠标位于终端或文件夹卡片内容区域
    When 用户执行缩放手势（Command+滚轮或触控板捏合）
    Then 应透传到画布并完成缩放，不被卡片内部滚动消费

16. Given 鼠标位于终端内容区并触发滚轮
    When 终端在当前手势内先消费滚动并随后触达边界
    Then 不应连带滚动画布（避免同手势链式透传）

17. Given 鼠标位于终端内容区并触发滚轮
    When 终端在手势开始前就处于对应边界
    Then 应将滚轮透传给画布

18. Given 用户右键终端卡片
    When 选择“切换为文件夹卡片”
    Then 当前卡片保持原位置与尺寸，仅类型切换为 `folder`

19. Given 用户右键文件夹卡片
    When 选择“切换为终端卡片”
    Then 当前卡片保持原位置与尺寸，仅类型切换为 `terminal`

20. Given 用户打开一个文件夹卡片
    When 卡片完成渲染
    Then 卡片内容区显示当前目录的文件与子目录列表

21. Given 用户在文件夹卡片中双击某个子目录
    When 目录切换完成
    Then 卡片进入该子目录，并刷新列表

22. Given 用户在文件夹卡片中点击“返回上级”
    When 当前目录存在父目录
    Then 卡片切换到父目录，并刷新列表

23. Given 用户打开文件夹卡片
    When 查看内容区的目录列表
    Then 列表背景、文字与图标应保持高对比可读，避免浅色背景与浅色文字叠加

24. Given 画布上已有多张卡片且位置凌乱
    When 用户触发“自动整理卡片”
    Then 卡片应按平衡网格重新排布且互不重叠

25. Given 用户选中一张或多张卡片
    When 用户点击画布右下角“对齐网格”按钮
    Then 目标卡片位置应吸附到网点网格坐标

26. Given 用户右键一个终端或文件夹卡片
    When 选择“展开为独立窗口”
    Then 应将卡片内同一内容视图迁移到独立窗口（非新建副本），画布卡片仅保留占位信息，且再次触发应激活已存在窗口而非重复创建
